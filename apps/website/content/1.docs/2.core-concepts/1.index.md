MiiaJS is built around a few core concepts that work together to create a clean, modular application architecture.

## Building blocks

| Concept | Description |
| --- | --- |
| [Controllers](/docs/core-concepts/controllers) | Handle incoming HTTP requests and return responses |
| [Routing](/docs/core-concepts/routing) | How URLs are matched - static, dynamic, wildcard routes |
| [Providers](/docs/core-concepts/providers) | Services, repositories, and other injectable classes |
| [Modules](/docs/core-concepts/modules) | Organize the application into cohesive blocks |
| [Configuration](/docs/core-concepts/configuration) | Manage environment variables and app settings |
| [Middleware](/docs/core-concepts/middleware) | Transform requests and responses in the pipeline |
| [Guards](/docs/core-concepts/guards) | Determine whether a request should be handled |
| [Exceptions](/docs/core-concepts/exceptions) | Handle errors with structured HTTP responses |
| [Response](/docs/core-concepts/response) | Build HTTP responses with a fluent API |
| [Testing](/docs/core-concepts/testing) | Test your application with the built-in TestApp |

## Request lifecycle

Every HTTP request in MiiaJS flows through a well-defined pipeline:

```
Request
  -> Global middleware
  -> Global guards
  -> Controller middleware
  -> Route guards
  -> Route middleware
  -> Validation (body/query/params)
  -> Route handler
  -> Response
```

Middleware follows a **Koa-style onion model** - each middleware can run code before *and* after the handler by wrapping `await next()`.
